# ✦ ShitGames

Плагин мини-игр для Paper: **ТнтРан**, **Сплиф** и **Царь горы**.
С полным сохранением инвентарей и защитой от багоюзов.

## Возможности

**ТнтРан**
Слои арены заполняются динамитом, сверху автоматически кладутся коврики. Наступил на коврик - через настраиваемую задержку коврик исчезает (без выпадения предметом), а динамит красиво падает вниз отдельной сущностью.

**Сплиф**
Слои заполняются снегом, каждому участнику выдаётся лопата. Блоки ломаются без дропа, соперники проваливаются этажом ниже.

**Общее для арен**
- Неограниченное количество арен, каждая настраивается отдельно.
- Голосование за мини-игру предметом `&eВыбор игры` через GUI-меню. Голос игрока, покинувшего арену до старта, не учитывается.
- Лобби-ожидание, отсчёт, укороченный отсчёт при полном наборе.
- Случайное распределение по точкам спавна: 1 точка = 1 игрок, при нехватке точек игроки распределяются по кругу.
- Боссбар `До сужения зоны: MM:SS`. Когда время вышло - `Время вышло!` и арена начинает принудительно разрушаться со всё возрастающей скоростью (защита от AFK).
- Случайная выдача предмета `&bДвойной прыжок` - ПКМ в воздухе даёт дополнительный прыжок.
- Победитель получает награду через настраиваемые консольные команды.
- Если на арене остался один игрок - он автоматически побеждает.
- **Все сообщения, звуки и частицы локальны**: их видят и слышат только участники арены.

**Царь горы**
Отдельная механика, не связанная с аренами. Админ встаёт на блок и сохраняет его как вершину. Кто простоит на вершине в одиночку нужное время - становится царём горы. Работает 24/7, текущему царю не показываются ни боссбар, ни сообщения, пока вершину не перехватят.

**Сохранность инвентаря**
При заходе на арену инвентарь, местоположение, режим игры, опыт, эффекты, здоровье и голод сохраняются в файл. Возврат гарантирован при выходе, выбывании, смерти, краше сервера, перезагрузке и перезапуске плагина.

## Установка

1. Положите `ShitGames-1.0.jar` в папку `plugins/`.
2. Запустите сервер - сгенерируется `plugins/ShitGames/config.yml`.
3. Настройте конфиг под себя и выполните `/sg reload`.

## Быстрая настройка арены

```
/sg wand
```
Вы получите золотой топорик. **ЛКМ** по блоку - первая точка выделения, **ПКМ** - вторая.

**Шаг 1. Создать арену.** Выделите весь объём арены целиком (все слои + пространство над и под ними) и выполните:
```
/sg create myarena
```

**Шаг 2. Добавить слои.** Выделите один слой (плоскую пластину блоков) и добавьте его. Повторите для каждого этажа - слоёв может быть сколько угодно:
```
/sg layer add myarena
```

**Шаг 3. Зона выбывания.** Выделите область под нижним слоем - упав в неё, игрок выбывает:
```
/sg deathzone myarena
```

**Шаг 4. Точка лобби.** Встаньте туда, где игроки будут ждать старта, и выполните:
```
/sg lobby myarena
```

**Шаг 5. Точки спавна.** Включите режим расстановки и топориком **ПКМ** по блокам - на каждом появится точка спавна (**ЛКМ** - убрать). Выключается повторной командой:
```
/sg spawns myarena
```

**Готово.** Проверьте настройку:
```
/sg info myarena
```
Игроки заходят командой `/sg join myarena` или `/sg join` (автовыбор арены). Вы можете добавить собственные алиасы команды '/sg' в `plugin.yml`

## Как строить арену

- **Регион арены** должен полностью включать все слои, коврики над ними и зону выбывания. По горизонтали игрок, вышедший за регион, выбывает.
- **Слой** - это плоское выделение блоков (обычно толщиной в 1 блок). Коврик для ТнтРана плагин ставит **сам** на один блок выше верхнего ряда слоя - не включайте место под коврик в выделение слоя.
- **Расстояние между слоями** - минимум 4 блока по вертикали: 1 блок слоя + 1 коврик + 2 блока роста игрока. Меньше - игроки будут застревать.
- **Зона выбывания**: важен её верхний Y. Всё, что на этом уровне и ниже, считается «упал». Делайте её пустой (воздух) под нижним слоем.
- **Лобби** можно ставить где угодно, хоть над ареной. Радиус, за который игрока не выпустят из лобби, задаётся `settings.lobby-radius`.
- Слои перед каждым раундом полностью восстанавливаются автоматически - вручную чинить арену не нужно.

## Царь горы

```
/sg hill set - встаньте на нужный блок и выполните команду
/sg hill clear - убрать вершину
/sg king info - текущий царь и координаты вершины
/sg king set <игрок>
/sg king clear
```

Игрок должен стоять на вершине **в одиночку**. Прогресс показывается боссбаром. Если кто-то встал рядом - захват приостанавливается. Время захвата, потеря прогресса, звуки и награды настраиваются в секции `king-of-the-hill`.

## Команды

Основная команда - `/shitgames`, короткий алиас - `/sg`. Все подкоманды имеют автодополнение.

### Игрокам

| Команда | Описание                                   |
|---|--------------------------------------------|
| `/sg join [арена]` | Зайти на арену (без аргумента - автовыбор) |
| `/sg leave` | Покинуть арену                             |
| `/sg list` | Список арен и их состояние                 |

### Администраторам

| Команда | Описание |
|---|---|
| `/sg wand` | Получить кисть выделения |
| `/sg create <имя>` | Создать арену из выделения |
| `/sg delete <имя>` | Удалить арену |
| `/sg info <имя>` | Подробная информация об арене |
| `/sg region <имя>` | Переустановить регион арены |
| `/sg layer add <имя>` | Добавить слой из выделения |
| `/sg layer remove <имя> <номер>` | Удалить слой по номеру |
| `/sg layer clear <имя>` | Удалить все слои |
| `/sg deathzone <имя>` | Установить зону выбывания |
| `/sg lobby <имя>` | Установить точку лобби |
| `/sg spawns <имя>` | Вкл/выкл режим расстановки точек кистью |
| `/sg spawn add <имя>` | Добавить точку спавна там, где вы стоите |
| `/sg spawn clear <имя>` | Удалить все точки спавна |
| `/sg limits <имя> <мин> <макс>` | Лимиты игроков для конкретной арены |
| `/sg toggle <имя>` | Включить/выключить арену |
| `/sg start <имя>` | Запустить раунд принудительно |
| `/sg stop <имя>` | Остановить раунд |
| `/sg hill set` / `/sg hill clear` | Управление вершиной горы |
| `/sg king set <игрок>` / `/sg king clear` / `/sg king info` | Управление Царём горы |
| `/sg reload` | Перезагрузить конфигурацию |

## Конфигурация

Настраивается **всё**: тексты, звуки, частицы, предметы, боссбары, титры, лимиты, тайминги, награды. Файл `config.yml` подробно закомментирован на русском.

**Цвета** - три формата в любой строке:
```yaml
name: "&eОбычные коды"
name: "&#FFD166HEX-цвет"
name: "<gradient:#FFD166:#FF6B6B>MiniMessage</gradient>"
```

**Звуки** - `ключ громкость высота`, `none` отключает:
```yaml
arena-join: "minecraft:entity.experience_orb.pickup 1.0 1.4"
arena-leave: "none"
```

**Частицы** - `ЧАСТИЦА количество сдвигX сдвигY сдвигZ скорость`:
```yaml
game-start: "FIREWORK 60 2.0 1.5 2.0 0.15"
```

**Награды** - любые консольные команды, плейсхолдеры `%player%`, `%uuid%`, `%arena%`, `%game%`:
```yaml
rewards:
  winner-commands:
    - "give %player% diamond 3"
    - "eco give %player% 500"
```

Ключевые секции: `settings`, `game` (лимиты, отсчёты, разрушение зоны), `games.tnt-run`, `games.spleef`, `double-jump`, `vote`, `bossbar`, `titles`, `rewards`, `king-of-the-hill`, `sounds`, `particles`, `messages`.

## Защита от багоюзов

Плагин закрывает следующие дыры «из коробки»:

- **Инвентарь.** Полностью очищается на входе, любые клики, перетаскивания, дропы, подборы предметов, обмен рук и открытие сундуков заблокированы. Открыть можно только меню голосования.
- **Режим игры.** Любая попытка сменить режим на арене отменяется - даже если её инициирует сторонний плагин. Полёт принудительно отключён.
- **Телепортация.** Все внешние телепорты с арены блокируются, `/spawn`, `/home`, `/tpa` и подобные команды недоступны (белый список настраивается).
- **Урон и голод.** Полностью отключены, чтобы игрока нельзя было убить или выбить с арены сторонней механикой.
- **Смерть и краш.** При смерти дроп очищается, точка респавна подменяется на исходную, инвентарь возвращается. При краше сервера инвентари лежат в файлах и восстанавливаются при следующем входе игрока.
- **Голосование.** Голос привязан к UUID и удаляется при выходе с арены. Подсчёт идёт только по игрокам, реально находящимся на арене на момент старта - накрутить нельзя.
- **Дроп блоков.** Коврики и снег не выпадают предметами ни при обвале, ни при разрушении лопатой, ни при принудительном разрушении.
- **Динамит.** Поджиг, взрывы, воспламенение и физика внутри регионов арен отключены - динамит остаётся декоративным.
- **Падающие блоки.** Не приземляются, не превращаются в блоки и не создают дроп, удаляются по таймеру и при сбросе арены.
- **Границы.** Выход за пределы региона во время игры = выбывание; в лобби игрока возвращает обратно.
- **Двойные заходы.** Игрок не может находиться на двух аренах, зайти в идущий раунд или превысить лимит.
- **Гриферство.** Блоки арен защищены от игроков без права `shitgames.admin` (отключается в `settings.protect-arenas`).
- **Grace-период** в начале раунда: блоки не проваливаются и не ломаются, пока все не приземлились.

## Производительность

- **Ни одного постоянного поллинга.** Основной цикл - одна задача раз в секунду на все арены сразу; она мгновенно выходит, если арен нет. Потиковая задача включается только на время активного раунда ТнтРана и сама себя выключает.
- **Событийная модель.** Зоны, коврики, вершина горы - всё на событиях с ранним выходом по сравнению блочных координат, а не по расстояниям.
- **Конфиг читается только при старте и `/sg reload`**, все значения (цвета, звуки, предметы, материалы) заранее разобраны в неизменяемые объекты и подменяются атомарно.
- **Adventure-компоненты сообщений кэшируются** при загрузке конфига, а не пересобираются при каждой отправке.
- **fastutil** для примитивных коллекций (очереди обвала, наборы позиций) - без боксинга.
- **Восстановление арены разбито по тикам** (`game.reset-blocks-per-tick`) и выполняется без физических апдейтов.
- **Файловые операции** (инвентари, данные царя) асинхронны, игровой поток не блокируется.
- **Шаблоны предметов** собираются один раз и клонируются.

## Файлы плагина

```
plugins/ShitGames/
├── config.yml # вся настройка
├── arenas.yml # сохранённые арены
├── king.yml # вершина горы и текущий царь
└── vaults/ # сохранённые инвентари игроков, находящихся на аренах
```

`vaults/` очищается автоматически: файл удаляется сразу после возврата инвентаря игроку. Если там лежат файлы - значит, эти игроки ещё не получили инвентарь обратно, **не удаляйте папку вручную**.
